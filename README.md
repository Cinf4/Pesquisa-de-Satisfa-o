## 🚀 Sobre o Projeto

O programa utiliza estruturas de repetição e decisão para coletar informações de 50 entrevistados, validando os dados de entrada e gerando um relatório final consolidado. Foi desenvolvido com foco em **Clean Code** (Código Limpo) e **Modularização**, facilitando a manutenção e a escalabilidade do sistema.

## 🛠️ Requisitos Técnicos (Baseado no Desafio)

- [x] Utilizar a estrutura de repetição `FOR` para coletar dados de 50 pessoas.
- [x] Solicitar Nome, Idade e Opinião (1: Excelente, 2: Bom, 3: Ruim).
- [x] Utilizar estruturas de decisão (`if/elif/else`) para processar os votos.
- [x] Exibir a quantidade total de respostas "Excelente" e "Ruim".
- [x] Validar o funcionamento com um teste de 10 entrevistados.

## ✨ Diferenciais e Inovações (Além do Requisito)

Para elevar a qualidade do projeto e demonstrar boas práticas de desenvolvimento, adicionei:
- **Tratamento de Exceções (`try/except`):** O sistema não trava se o usuário digitar letras onde deveriam ser números.
- **Modularização:** O código é dividido em funções específicas para validação e exibição, seguindo o princípio de responsabilidade única.
- **Interface Amigável:** Menus formatados no terminal para uma melhor experiência do usuário (UX).
- **Validação de Dados:** Uso de laços `while` para garantir que a idade e a opção escolhida sejam válidas antes de prosseguir.
